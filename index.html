Problem

「自分で勉強しているのに、
なぜか伸びない」

  • 😮 YouTubeで見て「わかった気」になっても、いざ自分で作れない
  • 😓 何週間も同じところで止まったまま、そのまま辞めていく
  • 🤔 何をやればいいかわからず、とりあえず検索しているだけになっている
  • 😱 エラーが出ても何を調べればいいかわからない——言葉にすらできない

これは意欲の問題でも、頭の問題でもありません。
「正しい方向に進んでいるか、確かめる仕組みがない」ことが原因です。

Agitation

自学は最強の学び方です。
でも、落とし穴があります。

  • 間違った方向に進んでいても、自分では気づけない。
    誰かに確認してもらう仕組みがないと、何ヶ月も遠回りを続ける。
  • 「何を聞けばいいか」がわからなければ、AIにも聞けない。
    壁打ちできない状態では、ネットにいくら情報があっても使えない。
  • ひとりで詰まり続けると、挫折する。
    スキルではなく「自分には向いていない」という誤った結論を出してしまう。
IT業界で伸びる人は、「自分で学べる人」です。
でも「自分で学べる人」になるためには、最初に正しく進む経験が必要です。
誰かと一緒に、正しい方向を確認しながら進んだ人が、やがてひとりで走れるようになる。

Solution

「自学」と「伴走」を
セットにした場が、ここです。

01

自分で進む

先生が「次はこれをやろう」とは言いません。今日何をやるかを自分で決めるところから始まります。手を動かした時間が、そのままスキルになります。

02

詰まったら即解決

現役エンジニアの先生が常駐しています。「何がわからないかわからない」状態でも、一緒に整理します。3分以内に応答します。

03

方向を毎回確認する

1時間の最後に必ず先生と1対1の時間を設けます。「正しく進んでいるか」「次は何をやるか」を毎回確かめます。迷子のまま終わりません。

04

聞き方を育てる

「〇〇をしようとして、△△を試したら□□になった」と言えるようにします。AIにもコミュニティにも正しく聞けるこの力が、一生の武器になります。

Curriculum

Scratchの次から、
本物のエンジニアの言語へ。

Scratchで培った「プログラミングの考え方」をそのまま活かして、 現場で使われるJavaScriptへ移行します。ゲーム制作を通じてWebアプリまで到達する、 一貫したカリキュラムです。

Stage 01

アプリ開発入門

HTMLの仕組み・デバッガーの使い方から。スマートフォンアプリとして動く画面を最初から作ります。

HTML / alert / デバッガー

Stage 02

HTML・CSS入門

Webページの構造とデザインを理解。リンク・画像・レイアウトを自分で組み立てられるようになります。

HTML / CSS / セレクタ

Stage 03

JavaScript基礎

変数・条件分岐・関数・イベントを習得。「今日の日付を取得する」など実用的なプログラムを作ります。

JS / 変数 / 関数 / DOM

Stage 04

データ構造・繰り返し

配列・ループ・演算子を使いこなし、複雑なロジックを自分で設計できるようになります。

配列 / for文 / 演算子

Stage 05

Canvas・ゲームプログラミング

ブラウザで動く本格ゲームを制作。15パズル・神経衰弱・シューティングなど自分の作品に仕上げます。

Canvas / ゲーム設計 / 実装

Stage 06

サンプルアプリ制作

ブロック崩しをベースに、ゼロから自分でアレンジした作品を完成させます。ポートフォリオになります。

アレンジ / 自走 / 発表

※ 各自のペースで進めます。全員が同じ進度である必要はありません。
※ Progate「JavaScript II」（有料版）を参考教材として使用する段階があります。

Graduation

ゴールは「卒業できること」です。

IT業界では、自分で学べる人が圧倒的に有利な世界です。
ネットにはいくらでも学べる環境がある。AIにも聞ける。でも、「何を・どう聞けばいいか」がわからなければ、何も使えない。

ITオンライン部の本当のゴールは、この状態を卒業すること。
下のようなIT系コミュニティの会話が「だいたいわかる」「調べれば進めそう」と感じられるようになったとき、このコースを卒業し、自学できる人になったということです。

IT系コミュニティで実際に飛び交う会話 ↓

A

Aさん

非同期処理で詰まってる。awaitつけたのに順番通りに動かなくて
B

Bさん

forEachの中でawaitしてない？それfor...ofに変えたら直るよ
A

Aさん

直った！あとエラーハンドリングってtry-catchでいい？
C

Cさん

本番で落ちると泣くから絶対書いて。ボトルネックの特定もしとくといい

今はこの会話が全部わからなくて当然です。でも、これが「なんとなくわかる」「わからなくても調べられそう」になったとき、ITオンライン部は卒業です。

1

用語で状況を説明できる

「なんか動かない」ではなく「条件分岐の中でループが止まっている」と言える。言葉にできると、検索もAIへの質問もできるようになる。

2

AIやコミュニティに正しく聞ける

「〇〇をしようとして、△△を試したら□□というエラーが出た」の型で聞ける。この聞き方ができる人は、どこへ行っても伸びていける。

3

上の会話の意味がだいたいわかる

全部知らなくていい。「これは調べればわかりそう」「これは聞けばいい」の判断ができる状態。知らないことを怖がらなくなること。

「聞き方を知っている人」は、
どこへ行っても伸びていける。

卒業できる子を育てることが、私たちの仕事です。 — IT自由研究室 代表 中島ゆき
